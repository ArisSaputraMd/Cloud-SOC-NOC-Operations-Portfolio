## Project Roadmap

#### 1. Project Vision

This project is designed to simulate the build-out of a cloud-based Security Operations environment from foundational infrastructure to detection engineering.

Rather than deploying isolated lab components, this portfolio follows a phased operational maturity model, similar to how real organizations implement cloud security capabilities over time.

The objective is to demonstrate hands-on capability in:

- Cloud infrastructure design
- Secure configuration management
- Network monitoring (NOC operations)
- Security monitoring (SOC operations)
- Detection engineering and threat simulation

#### 2. Execution Strategy

The project is divided into four structured phases:

- Foundation (Secure Cloud Baseline)
- Cloud NOC Operations
- SOC Operations
- Detection Engineering

> _Each phase builds on the previous one, progressively increasing operational complexity and security depth._

#### 3. Phase Breakdown

**Phase 1 – Foundation**

Objective: Establish a secure and controlled AWS baseline.

Scope:

- AWS account security hardening
- IAM role-based access control model
- VPC design and subnet segmentation
- Security Groups and NACL baseline
- CloudTrail logging enablement
- Cost monitoring configuration

Deliverable:

- Documented secure cloud architecture
- Baseline logging enabled
- Initial network topology diagram

**Phase 2 – Cloud NOC Operations**

Objective: Implement infrastructure health and availability monitoring.

Scope:

- Resource health monitoring
- CloudWatch metrics and alarms
- Availability validation
- Incident response workflow (infrastructure outages)
- Alert documentation

Deliverable:

- Monitoring dashboards
- Alerting strategy documentation
- Simulated outage analysis

**Phase 3 – SOC Operations**

Objective: Build centralized logging and security monitoring.

Scope:

- Log aggregation
- Security event monitoring
- Alert triage workflow
- Incident documentation process
- Threat simulation exercises

Deliverable:

- Security monitoring documentation
- Incident case studies
- Detection validation reports

**Phase 4 – Detection Engineering**

Objective: Develop custom detection capabilities and validate security controls.

Scope:

- Log-based detection use cases
- Threat hunting methodology
- MITRE ATT&CK mapping
- Attack simulation and validation
- Continuous improvement model

Deliverable:

- Detection rule documentation
- Threat simulation reports
- Detection improvement analysis

**4. Milestone Strategy**

This project is designed to allow incremental portfolio value.

- Completion of Phase 1 enables readiness for entry-level Cloud/NOC roles.
- Completion of Phase 2 demonstrates operational monitoring capability.
- Completion of Phase 3 positions for SOC Analyst roles.
- Completion of Phase 4 demonstrates detection engineering fundamentals.

**5. Long-Term Objective**

By completing this portfolio, the goal is to demonstrate the ability to:

- Design secure cloud infrastructure
- Operate and monitor production-like systems
- Detect and respond to security events
- Improve detection capability over time
